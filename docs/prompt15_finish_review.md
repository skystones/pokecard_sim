# Prompt 15: 仕上げレビュー用プロンプト（改訂版）

ある程度実装が進んだ後、Codexに以下を渡してください。

```text
これまでの実装をレビューしてください。

観点:
1. GameStateとObservationが分離されているか。
2. GameStateがPlayerStateを通じて相手の場・相手デッキ・相手手札に拡張できるか。
3. Agentが山札順やサイド内容を不正に見ていないか。
4. 乱数seedで再現可能か。
5. card_defs.yamlのreviewed=false / guessed=true / source_urls / notesが適切に管理されているか。
6. 未実装カードがないか（implementedフラグや同等の実装管理情報とコード実体を突合し、残件がある場合は一覧と理由を示す）。
7. 目的関数の順序条件が正しく判定されているか。
8. ハード条件とソフト条件が辞書式順序で評価されているか。
9. ログが失敗要因分析に十分な情報を含んでいるか。
10. 相手ターン・相手の場を将来実装する余地があるか。
11. RL環境にaction maskがあるか。
12. RLの評価がhard_success_rate基準になっているか（soft指標は補助で分離表示されているか）。
13. pytestが十分か（特に辞書式順序と目的関数判定をカバーしているか）。
14. belief sampling / MCTS / imitation learning / RLへ拡張できる構造か。

問題があれば、修正案を出してから実装してください。
破壊的変更が必要な場合は、理由を説明してください。

出力形式:
1) 指摘一覧（各項目に severity: blocker/high/medium/low と根拠）
2) 修正方針（破壊的変更の有無・理由）
3) 実装修正
4) 追加/更新テスト一覧
5) 再レビュー結果（観点1〜14のチェック表）

必須:
- 辞書式順序（hard優先）のテストを追加すること。
- hard失敗かつsoft高得点でも失敗判定になるテストを追加すること。
- action mask妥当性とhard_success_rate評価をログで確認すること。
- 未実装カードが残る場合は一覧と理由を明示すること。
```

# 追加メモ: Codexに強く守らせたい禁止事項（改訂版）

```text
禁止事項:
- 設計注意事項を読まずに実装を始めない。
- GameStateを自分専用のsingle-player構造に固定しない。
- 相手の場を後から追加できない構造にしない。
- カード効果を曖昧なまま確定実装しない。
- Agentに完全な山札順やサイド内容を渡さない。
- 未実装効果を何もせず成功扱いにしない。
- objectiveとrulesを混同しない。
  - 例: にせオーキド後のエリカ使用はルール違反ではなく、今回の目的条件違反。
- 最終的にRL実装をplaceholderのまま放置しない。
- ログなしで確率だけ出さない。
- seed非管理の乱数を使わない。
- テストなしでカード効果を追加しない。
```

## 推奨ログ最低項目

- seed
- episode_id
- turn
- player_id
- selected_action
- action_mask可否（invalid判定理由を含む）
- hard/soft判定内訳
- failure reason code（列挙値）
